# 家庭OA审批管理系统 — 开发需求文档（单页Web应用）

## 一、技术方案

使用 **纯前端单页应用（Single Page Application）**，仅一个 HTML 文件（内嵌 CSS + JavaScript），不依赖任何后端服务器、数据库、第三方 API、无框架构建工具。

**数据存储方案：** 浏览器 `localStorage`。
- 用户数据（3个账号）预置在代码中，不提供注册修改入口
- 审批单据数据全部序列化为 JSON 存入 localStorage
- 每次操作（提交、审批、重新提交）立即写入 localStorage
- 页面刷新后从 localStorage 恢复全部状态

所有日期时间使用浏览器本地时间（`new Date().toLocaleString('zh-CN')`）。

## 二、用户系统

### 2.1 预置账号（写死在代码中，无注册入口）

| 账号 | 密码 | 显示名称 |
| admin1 | admin123 | 管理员一 |
| admin2 | admin123 | 管理员二 |

### 2.2 权限模型

- 两位管理员**完全平权**，无上下级区分
- 任意管理员均可：发起申请、审批任何单据、查看全部历史
- 没有"超级管理员"概念

## 三、登录页面

### 3.1 页面结构

登录页提供 **2个独立的账号入口卡片**，每个卡片展示：
- 账号图标/头像区（可用 emoji 或纯色圆形头像）
- 账号名称（显示名称 → 管理员一/二/三）
- 密码输入框（仅密码，账号名固定不可改）
- 登录按钮

三个卡片横向排列，居中展示。背景简洁。

### 3.2 登录逻辑

- 点击登录按钮 → 验证密码是否正确
- 密码正确 → 跳转主页面，会话状态通过 localStorage 记录当前登录人
- 密码错误 → 在对应卡片下方显示「密码错误」提示
- 登录后无过期机制，关闭页面重开需重新登录

## 四、主页面布局

登录后进入主页面，顶部显示当前登录人名称和「退出登录」按钮。

主页面分**三个区域**，用 Tab/卡片切换或上下分区均可：

### 4.1 发起新申请（表单区）

表单字段及校验规则：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| OA需求描述 | 文本域 | 是 | placeholder: "请描述需要审批的事项" |
| 项目类别 | 下拉选择 | 是 | 选项：办公用品采购 / 家庭维修 / 差旅报销 / 礼金礼赠 / 其他 |
| 预算金额 | 数字输入 | 是 | 单位：元，placeholder: "请输入金额" |
| 执行时间 | 日期选择器 | 是 | 格式：YYYY-MM-DD |
| 申请理由 | 文本域 | 是 | placeholder: "请详细说明申请理由" |

**交互规则：**
- 任意字段为空时，提交按钮置灰不可点击
- 提交成功后清空表单，弹窗提示「提交成功，等待另一位管理员审批」
- 单据ID自动生成：格式 `OA-YYYYMMDD-XXX`（XXX 为当天第几张单的序号，从001开始）

### 4.2 待我审批列表

- 仅展示**当前登录人尚未审批**且**状态为待审批**的单据
- 如果当前账号是该单据的发起人，则该单据不出现在待审批列表中（不能自己审批自己）
- 每张单据以卡片形式展示全部表单内容、发起人、发起时间

每个待审批卡片底部提供：

**① 同意+附注**
- 文本输入框：placeholder "填写审批意见（选填）"
- 「同意」按钮 → 点击后记录：操作人、操作时间、附注内容、操作类型=「同意」
- 如果当前登录人已是该单据的**第二个审批通过的人** → 单据状态变为「全部通过」
- 如果当前登录人是第一个审批通过的人 → 状态仍为「待审批」，等待第三人

**② 拒绝+附注**
- 文本输入框：placeholder "填写驳回理由"
- 「拒绝」按钮 → 点击后记录：操作人、操作时间、附注内容、操作类型=「拒绝」
- 拒绝后单据状态立即变为「已驳回」，**无需等待第二个审批人**
- 驳回后该单据从所有待审批列表中移除

**审批状态流转规则：**

```
发起申请 → 待审批（等待另一人审批）
   ├── 对方同意 → 状态：全部通过 ✓
   └── 对方拒绝 → 状态：已驳回 ✗
```

- 每个单据记录已审批人列表，同一人不可重复审批同一单据
- 页面顶部显示数字徽章：当前登录人「待审批 (N) 条」

### 4.3 全部历史审批记录

#### 展示规则
- 分页展示**所有单据**（含待审批、全部通过、已驳回三种状态）
- 每页 **10 条**
- 底部翻页控件：上一页 / 页码列表 / 下一页
- 排序规则：**按发起时间倒序**（最新在最前）

#### 每条记录展示内容

以卡片或表格行展示：

| 展示项 | 内容 |
|--------|------|
| 单据编号 | OA-YYYYMMDD-XXX |
| 发起人 | 管理员一/二/三 |
| OA需求描述 | 全文 |
| 项目类别 | 所选类别 |
| 预算金额 | ¥ 1,234.00 格式显示 |
| 执行时间 | YYYY-MM-DD |
| 申请理由 | 全文 |
| 最终状态 | 标签展示：待审批 🟡 / 全部通过 🟢 / 已驳回 🔴 |
| 审批日志 | 时间线形式展示每一步（如有） |

#### 审批日志时间线格式

每条审批记录按操作时间正序排列：

```
[2026-06-29 14:30:00] 管理员一 → 发起申请
[2026-06-29 15:00:00] 管理员二 → 同意  附注：预算合理，同意执行
[2026-06-29 15:30:00] 管理员三 → 同意  附注：无异议
                         最终状态：全部通过 ✓
```

或拒绝场景：

```
[2026-06-29 14:30:00] 管理员一 → 发起申请
[2026-06-29 15:00:00] 管理员二 → 拒绝  附注：预算超支，建议调整后重新提交
                         最终状态：已驳回 ✗
```

#### 再次提交功能

仅当以下条件全部满足时，在已驳回单据上显示「再次提交」按钮：
- 当前登录人 = 该单据的发起人
- 单据状态 = 已驳回

点击「再次提交」→ 弹出一个表单，预填原单据的全部内容（可修改），修改后点击提交：

- 生成**新的单据**（新ID，新时间戳）
- 审批人记录全部清空（需要重新走另一人审批流程）
- 原驳回单据**保留不变**，作为历史记录永久存在
- 新单据状态 = 待审批

## 五、数据模型（localStorage JSON结构）

```javascript
// 用户数据（写死在代码中）
const USERS = {
  admin1: { name: '管理员一', password: 'admin123' },
  admin2: { name: '管理员二', password: 'admin123' },
  admin3: { name: '管理员三', password: 'admin123' }
}

// 单据数据（localStorage key: 'oa_approvals'）
// 数组，每个元素：
{
  id: 'OA-20260629-001',     // 单据编号
  creator: 'admin1',          // 发起人账号
  creatorName: '管理员一',     // 发起人显示名
    
  // 表单内容
  description: '采购打印机一台',   // OA需求描述
  category: '办公用品采购',        // 项目类别
  budget: 1500,                   // 预算金额（数字）
  executeDate: '2026-07-15',      // 执行时间
  reason: '现有打印机故障，需更换',  // 申请理由

  // 状态
  status: 'pending',       // pending | approved | rejected
  
  // 审批记录（时间线）
  timeline: [
    {
      type: 'submit',           // submit | approve | reject
      operator: 'admin1',       // 操作人账号
      operatorName: '管理员一',  // 操作人显示名
      time: '2026/6/29 14:30:00',  // 操作时间
      note: ''                 // 附注内容
    },
    // 审批人操作记录追加在此
  ],
  
  // 已审批人列表（用于去重判断）
  approvedBy: ['admin2'],       // 已同意审批的账号列表
  rejectedBy: [],               // 已拒绝审批的账号列表（仅用于日志展示）
  
  createdAt: '2026/6/29 14:30:00'  // 发起时间
}
```

## 六、UI/UX 设计规范

### 6.1 整体风格
- 简洁、轻量、商务风
- 主色：#1890ff（Ant Design 蓝）
- 成功绿：#52c41a
- 失败红：#ff4d4f
- 警告黄：#faad14
- 字体：系统默认无衬线字体
- 圆角：6px
- 卡片阴影：浅色 `box-shadow`

### 6.2 响应式
- 主要适配桌面浏览器（宽度 ≥ 1024px）
- 卡片区域最大宽度 960px，居中

### 6.3 组件要求
- 按钮：hover 效果
- 表单输入框：focus 显示边框高亮
- 状态标签：彩色圆角标签
- 时间线：左侧竖线 + 圆点 + 文字
- 分页控件：当前页高亮
- Toast 提示：提交/审批操作后弹出轻提示（2秒自动消失）

---

## 七、数据安全与完整性

1. **不可删除**：所有单据一经创建永久存在 localStorage，不提供删除按钮和删除 API
2. **不可篡改**：每步审批操作仅追加 timeline，不修改已有记录内容
3. **初始化**：首次加载时检测 localStorage 是否包含 `oa_approvals` 键，不存在则初始化为空数组 `[]`
4. **数据导出（可选加分项）**：历史记录区域底部提供「导出全部记录」按钮，将全部单据导出为 JSON 文件下载
